剑指 Offer 11. 旋转数组的最小数字

题目描述

把一个数组最开始的若干个元素搬到数组的末尾，我们称之为数组的旋转。输入一个递增排序的数组的一个旋转，输出旋转数组的最小元素。例如，数组 [3,4,5,1,2] 为 [1,2,3,4,5] 的一个旋转，该数组的最小值为1。  

示例 1：

输入：[3,4,5,1,2]
输出：1
示例 2：

输入：[2,2,2,0,1]
输出：0

思路

一般来说，对于数组中排序啊、找目的元素啊我觉得最常见的就是二分法。这个题也是，但是这个其中有两个坑是大多数人第一次做这个题选择的忽视了的。我将在下面的步骤讲出来。

首先二分法，我们就得收缩我们的的区间，那么我们先根据索引取个中间值，但是中间取到了，和谁比较啊？（问题一）

我们假如和左边比，比左边的大，那旋转的值肯定不是在左边，因为是有序的，旋转点是分界点，所以如果中间值还是比左边的大的话，说明左边这一部分是有序的，那么left移动到mid的位置，当小于它的时候，那就right等于mid。

我们假如和右边比较的话，好像这个逻辑也是成立的，那是随便和哪个比较都可以吗？

我们现在来看一下当12345的时候，如果我们和左边一比较，点移动到了右边，于是乎就找不到这个点1了。
这个时候我知道又有小可爱说那我和右边的比，54321， 一比较大于，那肯定就把左边的left继续往右移动。

这样的时候涉及到数组，最后就和右边的，或者倒序，总有意想不到的惊喜。

第二个点就是什么时候停止下来，left==right,遇见这样的数组[1, 0, 1, 1, 1]，旋转点 x = 1，这样其实结束的时候不是真的所在的位置，但是通过一番推理计算，可推出 nums[i] = nums[m]nums[i]=nums[m] ，且区间 [i, m][i,m] 内所有元素值相等，即有：
nums[i] = nums[i+1] = … = nums[m] = nums[x]
nums[i]=nums[i+1]=⋯=nums[m]=nums[x]

返回值能保证正确性，可能对应的索引位置是不对的，但是咱们道题只要求返回那个旋转点的值是多少，所以这个停止条件也是ok的。

以上的具体分析点击这里哦，反正我看了，我不会再看了哈哈哈

实际上，当出现 nums[m] = nums[j]时，一定有区间 [i, m] 内所有元素相等 或 区间 [m, j] 内所有元素相等（或两者皆满足）。对于寻找此类数组的最小值问题，可直接放弃二分查找，而使用线性查找替代。

当nums[mid]>nums[right]，这个时候就是你不知道到底这里是不是反转点，所以这个时候我们继续将右边的往中间移动一下，理由就是点击这里———

最后当left==right的时候，就时候停下来了，这个时候返回即可。left或者right都可以啊，因为这个时候相等了嘛、
