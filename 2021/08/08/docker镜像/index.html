什么是docker？什么是镜像？镜像与容器的区别？

一台物理机器虚拟化出很多个拥有完整操作系统，并且相互独立的“虚拟计算机”

虚拟化技术：全虚拟化 + 操作系统（OS）虚拟化

全虚拟化比如VMware 而Docker则是基于操作系统虚拟化的一种实现。

首先docker 三个关键点—》镜像、容器、仓库

第一我先讲一下基础的概念—

Docker是一个容器化平台，它以容器的形式将你的应用程序及所有的依赖项打包在一起，以确保你的应用程序在任何环境中无缝运行。

仓库是存放镜像的场所，Repository是同一类Docker镜像的集合，包含了不同tag的Docker镜像，就是一种存储、管理、分发镜像的一种应用服务。。

镜像就是运行容器的前提，它包含了我容器运行时候所需要的程序、库、资源、配置文件等一些东西。

Docker是一个容器化平台，它以容器的形式将你的应用程序及所有的依赖项打包在一起，以确保你的应用程序在任何环境中无缝运行。

然后我细讲一下镜像—

–镜像不包含任何动态数据，其内容在构建之后也不会被改变。镜像是一个固定的文件，这次读盘和下次读盘内容是一样的。镜像是一种分层的结构，Docker镜像是分层构建的，Dockerfile 中每条指令都会新建一层。

容器

事实上，容器（container）和镜像（image）的最主要区别就是在镜像加了顶层的读写层。所有对容器的修改都发生在此层，镜像并不会被修改，也就是经常提到的 COW(copy-on-write)技术。

容器需要读取某个文件时，直接从底部只读层去读即可，而如果需要修改某文件，则将该文件拷贝到顶部读写层进行修改，只读层保持不变。

每个容器都有自己的读写层，因此多个容器可以使用同一个镜像，另外容器被删除时，其对应的读写层也会被删除（如果你希望多个容器共享或者持久化数据，可以使用 Docker volume）。

最后，执行命令 docker ps -s，可以看到最后有两列 size 和 virtual size。其中 size就是容器读写层占用的磁盘空间，而 virtual size 就是读写层加上对应只读层所占用的磁盘空间。如果两个容器是从同一个镜像创建，那么只读层就是 100%共享，即使不是从同一镜像创建，其镜像仍然可能共享部分只读层（如一个镜像是基于另一个创建）。因此，docker 实际占用的磁盘空间远远小于 virtual size 的总和。

这也是因为一个Docker镜像被划分为多个镜像层layer。每一个layer其实都有唯一的标识diffid，基于layer内容经过sha256计算得出，且可以通过diffid得到layer的索引ID，实现按内容寻址。也因为Docker镜像分层的特点，可以实现不同镜像间共享相同layer，避免重复下载，节约资源，提升效率。

每个层其实是有一个指针指向自己的下一个层，这些层的交互、管理就需要存储驱动程序，也即联合文件系统（UnionFS）。

跳出原理

再回到应用方面，那么再想到我们如果运行一个软件，我运行软件就不用说根据不同的操作系统环境而需要重新配置，镜像不经包含了软件，还包含了我所需要的环境变量等等，我想也也是一次编译，四处运行的口号的意义之一嘛，这也是很基础的应用，但也是最接近我的。
摆脱掉系统对我们的限制，这样的部署方式，这样的隔离技术使得Docker成为了一个很厉害的技术。在大集群的部署，结合K8s的容器自动化管理，将会给我们整个生产带来很多的遍历和帮助。
